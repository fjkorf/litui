# Bevy ECS Integration for State Management

## Current Implementation (Path A: eframe + standalone bevy_ecs)

The eframe demo (`examples/eframe_demo/`) uses `bevy_ecs` 0.18 as a standalone state management library inside eframe. The ECS World lives inside the `DemoApp` struct.

```rust
pub struct DemoApp {
    world: World,      // owns all state as Resources
    schedule: Schedule, // runs systems each frame
}
```

Each frame: `self.schedule.run(&mut self.world)` ticks ECS systems, then render functions access state. Mutable pages use `world.resource_scope()`, read-only pages (Monitor) use `world.resource::<AppState>()`.

### Key Types

- `AppState` — generated by macro, holds all widget fields across all pages. `impl Resource for AppState {}` in demo_app.rs.
- `Page` — generated enum. `impl Resource for Page {}` in demo_app.rs.
- `auto_unmute` — example ECS system: forces `muted = false` when `volume > 80.0`.

### Why bevy_ecs

- Cross-cutting state access: the Monitor page reads `AppState` to display live values from the Form page
- Business logic as systems: `auto_unmute` runs every frame independently of which page is active
- Lightweight: `bevy_ecs` alone is ~2MB compile, no Bevy renderer/windowing

## Path B: Full Bevy (implemented as `bevy_demo` example)

The `examples/bevy_demo/` app renders the same 7-page demo inside a full Bevy application using `bevy_egui 0.39`. Compatibility confirmed: egui 0.33, bevy_ecs 0.18, wgpu 27.0 — all exact matches.

### Architecture

```
bevy_demo (Bevy binary)
├── define_markdown_app! { ... }  ← same content .md files as eframe_demo
├── Page and AppState as Bevy Resources
├── auto_unmute system (ResMut<AppState>) — identical to eframe version
├── render_nav system (EguiContexts, ResMut<Page>) — top panel navigation
└── render_page system (EguiContexts, Res<Page>, ResMut<AppState>) — page dispatch
```

### Key findings

- litui macro-generated code (`render_*()` functions) works **unchanged** inside `bevy_egui`'s egui context
- No changes needed to the proc-macro, helpers, or content files
- First compile: ~51s (bevy + all deps). Incremental: ~1s
- `EguiPrimaryContextPass` schedule label for UI systems; `.chain()` for ordering nav before page
- `egui` crate must be an explicit dependency (macro-generated code references `egui::` directly)

### Running

```sh
cargo run -p bevy_demo    # full Bevy version
cargo run -p eframe_demo  # eframe version (same content)
```

## ECS Primer (Bevy 0.18)

- **World**: container for all ECS data
- **Resource** (`#[derive(Resource)]`): singleton state (one instance per type)
- **System**: plain `fn` with typed parameters (`Res<T>`, `ResMut<T>`)
- **Schedule**: ordered collection of systems
