# Bevy Integration

> Run it: `cargo run -p bevy_demo`

litui works in full Bevy apps via `bevy_egui`. The generated `render_*()` functions only need
`&mut egui::Ui` -- they don't care whether that `Ui` came from eframe or Bevy.

## Dependencies

```toml
[dependencies]
bevy = { version = "0.18", default-features = false, features = [
    "bevy_winit",
    "bevy_render",
    "bevy_core_pipeline",
    "bevy_pbr",
    "x11",
] }
bevy_egui = "0.39"
egui = "0.33"
litui = { path = "../../crates/litui" }
```

`bevy_egui` provides `EguiPlugin`, which creates an egui context inside Bevy's render loop.
You get `&mut egui::Ui` through `EguiContexts` the same way you would through eframe --
panels, windows, areas all work identically.

## Getting a Ui

```rust,ignore
use bevy_egui::{EguiContexts, egui};

fn my_system(mut contexts: EguiContexts<'_, '_>) -> Result {
    egui::CentralPanel::default().show(contexts.ctx_mut()?, |ui| {
        // ui is &mut egui::Ui -- pass it to any litui render function
    });
    Ok(())
}
```

Every litui render function accepts `&mut egui::Ui`. Nothing changes.

## Page and AppState as Resources

`define_markdown_app!` generates `Page` and `AppState`. To use them in Bevy systems, impl
`Resource` and register them with the app:

```rust,ignore
use bevy::prelude::*;

impl Resource for Page {}
impl Resource for AppState {}
```

Then in your app setup:

```rust,ignore
App::new()
    .init_resource::<Page>()
    .init_resource::<AppState>()
```

Both types derive `Default`, so `init_resource` works out of the box.

## Systems

Split rendering into two systems: one for navigation, one for the page content. Use `.chain()`
to guarantee the nav panel renders before the central panel.

```rust,ignore
.add_systems(EguiPrimaryContextPass, (render_nav, render_page).chain())
```

`EguiPrimaryContextPass` is the schedule label where `bevy_egui` makes the egui context
available. All UI systems go here.

## Business logic systems

ECS systems that mutate `AppState` run in the normal `Update` schedule. They work identically
to the eframe version -- the only difference is Bevy owns the schedule.

```rust,ignore
.add_systems(Update, auto_unmute)

fn auto_unmute(mut state: ResMut<'_, AppState>) {
    if state.volume > 80.0 {
        state.muted = false;
    }
}
```

This runs every frame regardless of which page is active, enforcing invariants across the
entire app state.

## Complete main.rs

This is the full `bevy_demo` -- about 60 lines including imports.

```rust,ignore
use bevy::prelude::*;
use bevy_egui::{EguiContexts, EguiPlugin, EguiPrimaryContextPass, egui};
use demo_content::{self, AppState, Page};

fn main() {
    App::new()
        .add_plugins(DefaultPlugins.set(WindowPlugin {
            primary_window: Some(Window {
                title: "litui Bevy Demo".into(),
                ..default()
            }),
            ..default()
        }))
        .add_plugins(EguiPlugin::default())
        .init_resource::<Page>()
        .init_resource::<AppState>()
        .add_systems(Startup, setup)
        .add_systems(Update, demo_content::auto_unmute)
        .add_systems(EguiPrimaryContextPass, (render_nav, render_page).chain())
        .run();
}

fn setup(mut commands: Commands<'_, '_>) {
    commands.spawn(Camera2d);
}

fn render_nav(mut contexts: EguiContexts<'_, '_>, mut current: ResMut<'_, Page>) -> Result {
    egui::TopBottomPanel::top("nav").show(contexts.ctx_mut()?, |ui| {
        ui.horizontal_wrapped(|ui| {
            ui.visuals_mut().button_frame = false;
            for &page in Page::ALL {
                if ui
                    .selectable_label(*current == page, page.label())
                    .clicked()
                {
                    *current = page;
                }
            }
        });
    });
    Ok(())
}

fn render_page(
    mut contexts: EguiContexts<'_, '_>,
    current: Res<'_, Page>,
    mut state: ResMut<'_, AppState>,
) -> Result {
    egui::CentralPanel::default().show(contexts.ctx_mut()?, |ui| {
        egui::ScrollArea::vertical().show(ui, |ui| match *current {
            Page::About => demo_content::render_about(ui),
            Page::Text => demo_content::render_text(ui),
            Page::Lists => demo_content::render_lists(ui),
            Page::Tables => demo_content::render_tables(ui),
            Page::Styles => demo_content::render_styles(ui),
            Page::Form => demo_content::render_form(ui, &mut state),
            Page::Monitor => demo_content::render_monitor(ui, &state),
        });
    });
    Ok(())
}
```

Key points:

- `Page::ALL` and `page.label()` are generated by the macro -- no manual enum wiring.
- `render_form` takes `&mut AppState` (mutable widgets), `render_monitor` takes `&AppState`
  (display-only). The macro decides which signature each page gets.
- The `demo_content` crate holds the `define_markdown_app!` call and the `Resource` impls.
  Both `bevy_demo` and `eframe_demo` share it.

## Build times

First compile pulls in all of Bevy -- expect around 50 seconds. Incremental rebuilds after
code changes take about 1 second. Use `default-features = false` on the `bevy` dependency and
enable only the features you need.

## Same output, different runtime

The rendered UI is pixel-identical to the eframe version (`cargo run -p eframe_demo`, see
[tutorial 07](crate::_tutorial::_07_multi_page_apps)). litui generates egui widget calls.
Whether those calls happen inside eframe's app loop or Bevy's ECS schedule makes no difference
to the output.
